## 1. 产品定位

### 1.1 产品概述

**产品名称**：AI 出题助手\
**产品定位**：面向 K12 教师的轻量级 AI 智能出题系统 \
**核心场景**：通过多轮对话梳理出题需求，自动生成结构化试卷并支持预览和导出

**目标用户**：

- 小学、初中、高中各学科教师
- 需要快速出题、组卷的一线教师
- 追求效率提升的教研人员

### 1.2 用户痛点

| 痛点          | 现状             | 期望         |
| ----------- | -------------- | ---------- |
| **出题耗时**    | 一份试卷需 2-3 小时   | 10-30 分钟完成 |
| **题目质量不稳定** | 依赖个人经验，难度不均    | AI 辅助，质量稳定 |
| **知识点覆盖难**  | 手动梳理，容易遗漏      | 自动匹配，全面覆盖  |
| **题库查找繁琐**  | 翻书、搜题库，效率低     | 一键检索，智能推荐  |
| **试卷格式排版**  | Word 手动排版，重复劳动 | 自动生成，一键导出  |

### 1.3 核心价值

- **自然语言交互**：老师用日常语言描述需求（如"帮我出一份初二数学期末试卷"），AI 自动提取结构化参数
- **智能参数校验**：AI 主动判断信息完整性，缺失时追问而非盲目生成
- **题库优先策略**：优先使用真题库（质量有保障），AI 仅补充缺口（10-20%）
- **多轮对话记忆**：跨轮次保留上下文，支持渐进式补充信息
- **一键组卷**：参数确认后自动生成题目数组，按题型分组排列
- **灵活调整**：支持增删改、拖拽排序、单题改编、重新生成

### 2.2 用户旅程（简化版）

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ ① 首页  │───▶│ ② 对话  │───▶│ ③ 确认  │───▶│ ④ 生成  │───▶│ ⑤ 预览  │
│ 输入需求 │    │ AI追问  │    │ 参数面板 │    │ 试卷   │    │ 编辑导出 │

  ↓

  ↓

    📚 数学 | 🎓 初二 | 📕 人教版 | 📖 第16-18章

    [展示参数确认面板]
  ↓
用户：[确认并生成]
  ↓
AI：✅ 试卷生成完成！共 28 道题
    [自动切换为分栏预览]
```

### 2.3 参数体系（核心概念）

#### 参数分层

| 层级       | 参数          | 说明         | 必填    |
| -------- | ----------- | ---------- | ----- |
| **核心身份** | 学科、年级       | 确定出题范围     | ✅     |
| **内容入口** | 教材章节 或 知识点  | 双入口机制      | ✅ 二选一 |
| **场景策略** | 场景、难度、题型、题量 | AI 推断或用户指定 | 部分可选  |

#### 双入口机制

- **路径A**：教材版本 → 章节 → 节（适合按教学进度出题）
- **路径B**：知识点自由输入（适合专题训练、跨教材出题）

#### 场景策略

| 场景   | 难度分布                     | 题型分布  | 题量范围    |
| ---- | ------------------------ | ----- | ------- |
| 课后作业 | 简单 50% / 中等 40% / 困难 10% | 多样化   | 5-20 题  |
| 单元测验 | 简单 30% / 中等 50% / 困难 20% | 解答题增多 | 10-30 题 |
| 期末考试 | 简单 20% / 中等 50% / 困难 30% | 解答题为主 | 20-45 题 |
| 考前复习 | 简单 10% / 中等 40% / 困难 50% | 综合题为主 | 10-30 题 |

**详细规则**：参见 [PARAMETER\_RULES.md](./PARAMETER_RULES.md)

## 5. 非功能性需求

### 5.1 性能要求

- **响应时间**：对话回复 < 2s，试卷生成 < 30s
- **并发支持**：支持 100+ 并发用户（MVP 阶段）
- **可用性**：系统可用率 ≥ 99%（单月）

### 5.2 安全性要求

- **数据隐私**：不记录教师个人身份信息（姓名、学校）
- **会话隔离**：不同会话数据互不干扰
- **AI 内容审核**：生成题目自动检查违规关键词（暴力、色情、政治敏感）
- **API 鉴权**：所有接口需 Token 认证（后续版本）

### 5.3 合规性要求

- **教育合规**：符合教育部课程标准
- **AI 标注**：AI 生成内容明确标注"AI 生成，仅供参考"
- **人工审核**：教师可人工修改所有 AI 生成内容
- **未成年人保护**：不处理学生个人信息

## 6. 约束条件

### 6.1 技术约束

- **LLM 依赖**：需稳定的 LLM API 服务（DeepSeek/Claude）
- **题库规模**：MVP 阶段题库规模 10 万+题（后续扩展至 100 万+）
- **计算资源**：需支持 Elasticsearch（2GB+ 内存）和 PostgreSQL

### 6.2 时间约束

- **MVP 交付**：2026-05-31（约 5 周）
- **核心功能优先**：对话出题 > 试卷生成 > 编辑导出

### 6.3 成本约束

- **LLM 成本**：控制在 ¥0.5/份试卷以内（主要是 AI 生成部分）
- **基础设施**：云服务器 + 数据库，月成本 < ¥2000（MVP）

## 7. 风险与应对

| 风险              | 影响        | 概率 | 应对措施                        |
| --------------- | --------- | -- | --------------------------- |
| **LLM API 不稳定** | 生成失败      | 中  | 多提供商备份 + 异常重试 + 熔断机制        |
| **参数提取准确率低**    | 用户体验差     | 中  | 迭代 Prompt + 增加测试用例 + A/B 测试 |
| **题库覆盖不足**      | AI 生成占比过高 | 高  | 优先扩充题库 + 优化检索策略             |
| **生成速度慢**       | 用户流失      | 中  | 优化流程 + 缓存 + 并发调用            |
| **AI 生成质量差**    | 教师不信任     | 中  | 明确标注来源 + 支持人工修改 + 质量校验      |
