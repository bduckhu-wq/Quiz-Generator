## 🎯 拆分原则

### 1. 按职责分离

| 文档类型 | 负责部门 | 更新频率 | 稳定性 |
|---------|---------|---------|--------|
| **产品文档** | 产品团队 | 低 | 高 |
| **技术文档** | 技术团队 | 低-中 | 高 |
| **项目管理** | 项目团队 | 高 | 低 |

### 2. 按读者对象

- **产品经理**：PRD + 交互设计 + 参数规则
- **设计师**：交互设计
- **前端开发**：交互设计 + API 规范 + 数据模型
- **后端开发**：架构 + API 规范 + 数据模型
- **教研人员**：参数规则 + Skill 配置
- **项目经理**：PRD + 路线图 + 变更记录

### 3. 按更新频率

| 文档 | 更新频率 | 触发条件 |
|-----|---------|---------|
| PRD.md | 低 | 功能级迭代 |
| INTERACTION_DESIGN.md | 中 | UI 优化 |
| PARAMETER_RULES.md | 低 | 策略调整 |
| ROADMAP.md | 高 | 每周/每月 |
| CHANGELOG.md | 高 | 每次发布 |
| DECISION_LOG.md | 中 | 重大决策 |
