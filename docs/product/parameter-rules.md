## 0. 完整决策流程（总览）

### 0.1 核心决策原则

**核心架构**：题库优先，AI补充
```
所有出题请求 → 先查题库 → 判断匹配度
  ├─ 完全匹配 → 直接使用题库
  ├─ 部分匹配 → 题库 + AI补充
  └─ 无匹配 → 完全AI生成（回退方案）
```

**决策顺序**（优先级从高到低）：
```
1. 参数提取与必要性判定（学科、年级、内容入口）
2. 识别场景类型（日常练习/正式考试/考前复习）
3. 应用场景策略（难度分布、题型分布、题量）
4. 题库查询（双层：完整试卷库 → 题目库）
5. 根据匹配度选择生成模式（题库/混合/AI生成）
6. 返回试卷
```

**关键原则**：
- ✅ **题库优先**：所有场景统一先查题库，AI只做补充
- ✅ **双层查询**：完整试卷库（正式考试）→ 题目库（所有场景）
- ✅ **透明展示**：明确标注每道题的来源（题库/AI生成）

### 0.2 端到端决策流程图

```
用户输入
  ↓
步骤1：LLM参数提取（6个维度）
  ├─ 基础维度：学科、年级
  ├─ 内容维度：章节、知识点
  ├─ 来源维度：时间、地区、考试类型
  ├─ 筛选维度：题型、难度
  ├─ 组合维度：模式、试卷数、采样数
  └─ 场景维度：场景、学期
  ↓
步骤2：基础参数完整性校验
  必须：学科 + 年级
  至少一个：内容维度 或 来源维度
  ├─ 满足 → 继续
  └─ 不满足 → 批量追问
  ↓
步骤3：场景识别与推断
  根据关键词识别：
  ├─ 日常练习（作业、练习、单元测验）
  ├─ 正式考试（月考、期中、期末、中考）
  └─ 考前复习（复习、冲刺、压轴题）
  ↓
步骤4：场景约束校验（后置检查）
  IF 期末考试 → 必须有章节信息
    ├─ 有章节 → 通过
    ├─ 有知识点 → 反向映射到章节
    └─ 全无 → 追问学期，自动匹配章节
  IF 日常练习 → 必须有章节或知识点
  IF 精选组卷 → 必须有来源维度参数
  ↓
步骤5：应用场景策略（自动补全）
  - 题量默认值
  - 难度分布
  - 题型分布
  - 教材版本默认"人教版"
  - 地区IP自动定位
  ↓
步骤6：查询构建与层级决策
  6.1 确定查询层级
    ├─ 有考试类型+时间+地区 → 完整试卷库
    ├─ 仅内容/筛选参数 → 题目库
    └─ 有组合策略 → 试卷库→题目库
  
  6.2 构建查询条件（动态组合）
    WHERE条件根据参数动态生成
  
  6.3 选择执行模式
    ├─ direct：单次查询
    ├─ merge：多试卷汇总
    ├─ sample：多试卷采样
    └─ compare：多地区并行
  ↓
步骤7：执行查询与结果处理
  7.1 执行数据库查询
    ├─ 完整试卷库查询
    └─ 题目库查询
  
  7.2 匹配度判定
    ├─ >= 需求×1.5 → 题库充足
    ├─ 需求×0.3~1.5 → 题库+AI补充
    └─ < 需求×0.3 → 完全AI生成
  
  7.3 结果后处理
    ├─ 去重（相似度>85%）
    ├─ 排序（题型分组+难度递增）
    ├─ 标注来源
    └─ 难度均衡采样
  ↓
步骤8：生成试卷并返回
  包含：模式标识、题目数组、来源说明
```

**核心机制**：
- 所有参数在LLM层统一提取（6个维度）
- 场景约束后置校验，不前置限制
- 查询构建支持动态参数组合
- 4种执行模式适配不同场景

### 0.3 参数体系速查表

| 参数维度 | 参数名 | 必要性 | 提取方式 | 示例 | 说明 |
|---------|-------|-------|---------|------|------|
| **基础维度** | 学科 | ✅ 必须 | LLM提取/追问 | 数学 | 不可推断，必须明确 |
|  | 年级 | ✅ 必须 | LLM提取/追问 | 初二 | 不可推断，必须明确 |
| **内容维度** | 章节 | ⚠️ 场景依赖 | LLM提取 | 第16章、第16-18章 | 期末考试必须 |
|  | 知识点 | ⚠️ 场景依赖 | LLM提取 | 二次根式化简 | 与章节可共存 |
| **来源维度** | 时间范围 | ❌ 可选 | LLM提取 | 2024年、近3年、去年 | 支持绝对/相对时间 |
|  | 地区 | ❌ 可选 | LLM提取/IP定位 | 北京、海淀区、各区 | 支持模糊/精确匹配 |
|  | 考试类型 | ❌ 可选 | LLM提取 | 期末、月考、中考真题 | 影响试卷库查询 |
|  | 来源质量 | ❌ 可选 | LLM提取 | 真题、名校、教研室 | 质量排序依据 |
| **筛选维度** | 题型 | ❌ 可选 | LLM提取 | 选择题、压轴题 | 题型过滤 |
|  | 难度 | ❌ 可选 | LLM提取/场景策略 | 简单、中等、困难 | 难度过滤 |
| **组合维度** | 试卷数量 | ❌ 可选 | LLM提取 | 3份、5套 | 默认3-5份 |
|  | 采样策略 | ❌ 可选 | LLM识别 | 每份抽5题、汇总、各出10道 | 决定执行模式 |
| **场景约束** | 出题场景 | ❌ 可选 | LLM推断 | 期末考试、日常练习 | 触发场景策略 |
|  | 学期 | ⚠️ 条件必须 | LLM提取/追问 | 下学期 | 期末考试必须 |

**最低门槛**：学科 + 年级 + 至少一个内容/来源维度参数

**场景约束规则**：
- 期末考试 → 必须有章节信息（如无，反向映射或追问）
- 日常练习 → 必须有章节或知识点
- 精选组卷 → 必须有来源维度参数（时间/地区/考试类型）

**追问策略**：
- 基础维度缺失 → 批量追问
- 内容/来源维度全缺 → 提供选项追问
- 场景约束不满足 → 智能建议补全

### 0.4 场景与内容入口的映射关系

| 场景 | 优先使用的入口 | 是否强制 | 说明 |
|-----|--------------|---------|------|
| **课后作业** | 路径A（教材章节）或 路径B（知识点） | ❌ 灵活 | 教师既可以按章节布置，也可以按知识点布置 |
| **单元测验** | 路径A（教材章节）| ⚠️ 推荐但不强制 | 优先用路径A，但如果用户提供知识点，可自动反向映射到章节 |
| **期中期末考试** | 路径A（教材章节）| ✅ **强制** | 必须使用教材章节，且自动推断学期范围 |
| **考前复习** | 路径B（知识点）| ⚠️ 推荐但不强制 | 优先推荐智能策略（高频考点/易错点），但也可以按章节复习 |

**关键逻辑**：
- 期中期末考试**必须使用路径A**，因为考试范围明确对应学期章节
- 考前复习**推荐使用路径B**，因为复习聚焦知识点而非章节顺序
- 其他场景灵活选择，以用户输入为准

### 0.4 关键场景特殊逻辑

#### **期中期末考试**（强制路径A）
- 必须追问：教材版本 + 学期
- 自动匹配章节范围（如"初二数学下学期期末" → 第16-20章）
- 不允许纯知识点路径

#### **考前复习**（推荐路径B）
- 优先提供智能推荐：🔥高频考点 / ⚠️易错点 / 🎯压轴题
- 允许用户自定义知识点或按章节复习

## 1. 参数完整定义

### 1.1 基础维度参数

| 参数 | 必要性 | 说明 | 示例 |
|-----|-------|------|------|
| subject | ✅ 必须 | 学科（9科） | 数学、语文、英语、物理、化学、生物、政治、历史、地理 |
| grade | ✅ 必须 | 年级 | 小学1-6年级、初一至高三 |

### 1.2 内容维度参数

| 参数 | 必要性 | 说明 | 示例 | 格式支持 |
|-----|-------|------|------|---------|
| chapter | ⚠️ 场景依赖 | 教材章节 | "第16章"、"第16-18章" | 单章/多章/单节 |
| textbook | ❌ 可选 | 教材版本 | 人教版、北师大版、苏教版 | 可默认"人教版" |
| section | ❌ 可选 | 课本节（精确控制） | "16.1 二次根式" | 可选，用于精确定位 |
| knowledge_points | ⚠️ 场景依赖 | 知识点数组 | ["二次根式化简", "分母有理化"] | 可与章节共存 |

### 1.3 来源维度参数（智能搜索）

| 参数 | 必要性 | 说明 | 示例 | 提取方式 |
|-----|-------|------|------|---------|
| time_range | ❌ 可选 | 时间范围 | 2024年、近3年、去年 | LLM解析（绝对/相对） |
| region | ❌ 可选 | 地区 | 北京、海淀区、各区 | LLM解析/IP定位 |
| exam_type | ❌ 可选 | 考试类型 | 期末、月考、中考真题 | LLM提取 |
| source_type | ❌ 可选 | 来源质量 | 真题、名校、教研室 | LLM提取 |

### 1.4 筛选维度参数

| 参数 | 必要性 | 说明 | 示例 |
|-----|-------|------|------|
| question_type | ❌ 可选 | 题型过滤 | 选择题、压轴题、计算题 |
| difficulty | ❌ 可选 | 难度 | easy、medium、hard |
| score_min | ❌ 可选 | 最低分值 | 10（用于筛选大题） |

### 1.5 组合策略参数

| 参数 | 必要性 | 说明 | 示例 |
|-----|-------|------|------|
| mode | ❌ 可选 | 组合模式 | direct、merge、sample、compare |
| paper_count | ❌ 可选 | 试卷数量 | 3份、5套 |
| sample_per_paper | ❌ 可选 | 每份试卷抽取数 | 每份5题 |
| dedup | ❌ 可选 | 是否去重 | true/false |
| max_questions | ❌ 可选 | 最大题目数 | 30 |

### 1.6 场景与补充参数

| 参数 | 必要性 | 说明 | 示例 |
|-----|-------|------|------|
| scene | ❌ 可选 | 出题场景 | practice、exam、review |
| semester | ⚠️ 条件必须 | 学期（期末考试必须） | 上学期、下学期 |
| question_count | ❌ 可选 | 题量（场景策略自动） | 根据场景默认值 |
| difficulty_dist | ❌ 可选 | 难度分布（场景策略自动） | {easy: 0.2, medium: 0.5, hard: 0.3} |

## 2. 智能搜索机制 ⭐核心

### 2.1 核心设计理念

**统一智能搜索：** 用户无需理解"路径"概念，所有参数可自由组合查询

**支持的表达方式：**
- ✅ 单一维度："第16章"、"二次根式化简"、"2024年北京期末"
- ✅ 多维度组合："第16章的去年真题"、"海淀区的二次根式压轴题"
- ✅ 复杂搜索："3份海淀月考各抽5题"、"汇总北京各区期末压轴题"

**技术架构：**
```
用户自然语言 → LLM参数提取 → 查询构建 → 数据库执行 → 结果处理
```

### 2.2 参数提取与标准化

#### **LLM参数提取规则**

用户输入经LLM解析后，提取为标准化参数。

**关键转换规则：**

| 用户表达 | 参数维度 | 标准化值 |
|---------|---------|---------|
| "第16章" | 内容维度-章节 | "第16章" |
| "二次根式" | 内容维度-知识点 | ["二次根式的概念", "二次根式化简", "分母有理化"] |
| "2024年" | 来源维度-时间 | year=2024 |
| "去年" | 来源维度-时间 | year=当前年份-1 |
| "近3年" | 来源维度-时间 | year>=当前年份-3 |
| "北京各区" | 来源维度-地区 | region LIKE '北京%' |
| "压轴题" | 筛选维度 | difficulty='hard' AND score>=10 |
| "汇总" | 组合维度-模式 | mode='merge' |
| "每份抽5题" | 组合维度-采样数 | sample_per_paper=5 |

### 2.3 典型应用场景

教师在实际工作中的复杂需求：

| 需求类型 | 用户表达 | 技术目标 |
|---------|---------|---------|
| **多卷融合** | "把2024年北京各区期末试卷的压轴题汇总成一套" | 跨试卷检索+题型筛选 |
| **精选组卷** | "找3份海淀区月考的选择题，每份抽5道" | 多试卷采样+题量控制 |
| **特定来源** | "用去年期末真题+今年一模的题组卷" | 年份+类型组合查询 |
| **难度定制** | "只要压轴题和易错题" | 难度+题型精准过滤 |
| **地区对比** | "北京、上海、广东的中考题各出10道" | 多地区并行查询 |
| **时间范围** | "近3年的函数综合题" | 时间窗口+知识点查询 |

### 2.4 查询构建与执行流程

#### **步骤1：确定查询层级**

根据参数决定查询目标：

| 条件 | 查询层级 | 说明 |
|-----|---------|------|
| 有考试类型+时间+地区 | **完整试卷库** | 优先查询历年真题试卷 |
| 仅有内容/筛选参数 | **题目库** | 直接查询题目 |
| 有"汇总"/"采样"策略 | **试卷库→题目库** | 先查试卷，再提取题目 |

#### **步骤2：构建查询条件（动态组合）**

查询条件根据参数动态生成：

| 场景 | 查询层级 | 关键条件 |
|-----|---------|---------|
| 单一维度 | 题目库 | 基础+内容维度 |
| 多维度组合 | 题目库 | 基础+内容+来源维度 |
| 复杂搜索 | 试卷库→题目库 | 先查试卷，再提取题目 |

#### **步骤3：执行策略模式**

根据 `combination_strategy.mode` 选择执行模式：

| 模式 | 说明 | 典型场景 | 执行逻辑 |
|-----|------|---------|---------|
| **direct** | 直接查询 | "第16章作业" | 单次查询，直接返回 |
| **merge** | 汇总模式 | "汇总北京各区期末压轴题" | 多份试卷→提取题目→去重→合并 |
| **sample** | 采样模式 | "3份月考各抽5题" | 多份试卷→分层采样→分组展示 |
| **compare** | 对比模式 | "北京、上海、广东各出10题" | 多地区并行查询→分区展示 |

### 2.5 智能搜索高级能力

AI需要从自由文本中提取以下关键信息：

#### **1. 相似度去重**
- 文本相似度 > 85% → 判定为重复
- 优先保留：真题 > 名校题 > 教研室题
- 算法：编辑距离 / 余弦相似度

#### **2. 难度均衡采样**
当"每份试卷抽X题"时，确保难度分布合理：
- 简单：中等：困难 = 2:5:3
- 避免全部抽取简单题或全部困难题

#### **3. 题目来源标注**
每道题标注清晰来源，便于教师追溯：
- 来源试卷标题和ID
- 原始题号
- 年份、地区、考试类型

#### **4. 智能排序**
组合试卷的题目排序策略：
- 题型分组：选择题 → 填空题 → 解答题
- 难度递增：每题型内从易到难
- 来源标注：题号旁显示"[2024北京期末第25题]"

#### **5. 容错与追问**

- 参数不完整 → 追问年份/地区/考试类型
- 结果过多（>30份）→ 建议筛选（名校优先/指定区县/限定年份）
- 搜索无果 → 建议扩大范围或切换AI生成模式

### 2.6 场景约束与参数校验

**核心思路：** 不是前置限定使用哪条路径，而是后置检查场景必需参数是否满足。

#### **约束规则**

| 场景 | 必须参数 | 处理方式 | 示例 |
|-----|---------|---------|------|
| **期末考试** | 必须有章节信息 | 如无章节→反向映射或追问 | 用户说"考二次函数"→映射到第19章→建议补充为全学期章节 |
| **日常练习** | 必须有章节或知识点 | 如全缺→追问 | "出点作业"→追问"内容？[章节][知识点]" |
| **精选组卷** | 必须有来源条件 | 时间/地区/考试类型至少一项 | "帮我组卷"→追问"来源？[年份][地区][考试类型]" |

#### **自动补全示例**

**案例1：知识点反向映射**  
期末考试提到知识点→反向映射到章节→建议补充为全学期章节

**案例2：混合查询参数补全**  
"第16章的去年真题"→同时提取内容维度和来源维度→无需追问，直接执行组合查询

## 3. 场景策略

### 3.1 三大场景定义

| 场景 | 说明 | 出题模式 | 内容范围 | 题量/难度特征 |
|------|------|---------|---------|-------------|
| **日常练习** | 课后作业、单元测验 | 题库优先→AI补充 | 单章节或单知识点 | 题量少、难度偏低 |

## 8. 题库查询机制 ★核心

### 8.1 核心原则

**题库优先，AI补充**

```
出题请求 → 先查题库 → 判断匹配度
  ├─ 完全匹配（≥需求×1.5）→ 直接使用题库
  ├─ 部分匹配（需求×0.3~1.5）→ 题库+AI补充
  └─ 无匹配（<需求×0.3）→ 完全AI生成
```

### 8.2 双层查询结构

| 题库类型 | 优先级 | 适用场景 |
|---------|-------|---------|
| 完整试卷库 | ⭐⭐⭐ 最高 | 正式考试（优先推荐历年真题） |
| 题目库 | ⭐⭐ 高 | 所有场景（自由组合） |

**查询流程**：
1. 正式考试 → 先查完整试卷库（有匹配则展示选择）
2. 无完整试卷或其他场景 → 查题目库
3. 题库不足 → AI补充或完全AI生成

### 8.3 题库数据结构

#### **完整试卷库**
- 试卷ID、标题、年份、地区
- 学科、年级、考试类型、学期、教材版本
- 章节范围、题目数量、总分、时长、难度等级
- 完整题目数组

#### **题目库** ⭐核心
- 题目ID、学科、年级、章节、知识点数组
- 题型、难度、内容、选项、答案、解析、分值
- 来源（真题/教材配套）、来源试卷ID、年份、地区

**关键设计**：
- 支持一题多知识点标签
- 标注来源质量（真题优先）
- 可追溯到原始试卷

### 8.4 题库精准匹配策略 ⭐核心

#### **多维度过滤流程**

**第1步：强制过滤（硬性条件）**
- 学科、年级、章节/知识点（必须完全匹配）
- 题型范围（需求包含的题型必须匹配）
- 难度范围（根据场景策略限定）

**第2步：场景类型映射**

根据用户场景优先匹配题目库的"类型"标签：

| 用户场景 | 优先匹配的题目类型 |
|---------|------------------|
| 日常练习（课后作业） | 课时练习 > 单元测试 > 假期作业 |
| 正式考试-单元测验 | 单元测试 > 月考 > 课时练习 |
| 正式考试-月考 | 月考 > 期中 > 单元测试 |
| 正式考试-期中考试 | 期中 > 月考 > 真题（历年期中） |
| 正式考试-期末考试 | 期末 > 中考真题 > 中考模拟 > 学业考试 |
| 考前复习 | 中考真题 > 中考模拟 > 专题练习 > 竞赛 |

**第3步：质量优先级排序**
1. 年份越近越优先（2024-2026 > 2020-2023 > 2019以前）
2. 来源质量：真题 > 名校题 > 教研室题 > 普通题

**第4步：难度分布匹配**

题库难度（5档）与产品难度（3档）映射关系：

| 题库难度 | 产品难度 | 查询策略 |
|---------|---------|---------|
| 容易 | 简单 | 需要"简单"时查询"容易"+"较易" |
| 较易 | 简单↔中等 |  |
| 一般 | 中等 | 需要"中等"时查询"较易"+"一般"+"较难" |
| 较难 | 中等↔困难 |  |
| 困难 | 困难 | 需要"困难"时查询"较难"+"困难" |

按场景策略要求的难度分布比例抽取题目。

**第5步：题型分布匹配**
- 统计筛选出的题目题型分布
- 对比场景策略要求的题型分布
- 按比例抽取各题型题目

**第6步：章节均衡（多章节场景）**
- 如需求覆盖"第16-20章"
- 保证每章至少2-3题
- 避免某一章占比过高

**第7步：地区匹配策略**

| 场景 | 地区匹配策略 |
|-----|------------|
| 期末考试 | 优先本地区 > 全国 |
| 中考复习 | 必须本省 > 邻近省份 > 全国 |
| 日常练习 | 地区无关 |

### 8.5 AI补充策略

当题库不足时，AI生成的补充题目必须：

1. **匹配相同参数**：学科、年级、章节、知识点
2. **匹配难度分布**：根据场景策略生成相应难度
3. **匹配题型分布**：优先补充题库中缺失的题型
4. **避免重复**：不与题库题目内容重复

**智能缺口分析示例**：

```
场景：期末考试（需求28题）
题库筛选结果：18题
  ├─ 选择题：6题（需求7题，缺1题）
  ├─ 填空题：4题（需求4题，充足✅）
  ├─ 简答题：5题（需求10题，缺5题）
  └─ 探究题：3题（需求7题，缺4题）

AI补充策略：
  生成10题，精准填补缺口：
    ├─ 选择题：1题（中等难度）
    ├─ 简答题：5题（中等3题 + 困难2题）
    └─ 探究题：4题（中等2题 + 困难2题）
  
  分布要求：
    - 难度符合场景策略（简单20% + 中等50% + 困难30%）
    - 章节均衡分布
    - 标注来源为"AI生成"，与题库题目区分
```

### 8.6 用户体验提示

**完整试卷库匹配**：
- 展示历年真题列表（3-5份）供选择
- 提供"不使用完整试卷，自由组合"选项

**题库充足**：
- 说明题目来源（如：2025年期末真题8题、教材配套7题）

**混合模式**：
- 明确标注题库题目数量+AI补充数量
- 说明AI补充已匹配难度和知识点

**完全AI生成**：
- 说明原因（题库匹配度过低）
- 强调生成标准（严格按场景策略）
