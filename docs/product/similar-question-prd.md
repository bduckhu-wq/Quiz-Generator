## 📋 方案概述

**功能名称**：原题截图举一反三  
**核心价值**：教师上传原题截图，系统自动识别题目内容并生成 3 道相似题，保持核心学习目标不变，变化表层维度（场景、数值、表述）  
**目标用户**：需要快速生成练习题的教师，特别是需要"同类题强化训练"的场景

---

## 1. 需求背景

### 1.1 用户痛点

| 痛点 | 现状 | 期望 |
|-----|------|------|
| **找相似题难** | 需要手动翻题库，找同类型题耗时 30 分钟+ | 上传原题秒级生成相似题 |
| **改编题费脑** | 自己改编题目容易改变难度或知识点 | AI 保持核心不变，精准改编 |
| **练习量不足** | 一道好题只能用一次，举一反三靠人工 | 基于一道题快速扩展 3-5 道 |
| **重复率控制** | 担心学生做到原题，需要手动调整 | 70% 相似度，场景陌生化，防止"套格式" |

### 1.2 使用场景

**场景 1：课后作业补充**  
教师从教材或试卷中挑选 1 道典型题，生成 3 道相似题作为课后练习，强化同一知识点。

**场景 2：错题强化训练**  
学生做错某道题，教师上传该题截图，生成 3 道相似题让学生再练，巩固薄弱环节。

**场景 3：考前刷题**  
教师上传历年真题，生成相似题作为考前模拟练习，避免学生背答案。

**场景 4：分层教学**  
同一道题生成不同场景的相似题，分配给不同班级，避免答案传播。

## 2. 功能设计

### 2.1 核心流程

```
用户上传截图 → OCR识别题目 → 生成3道相似题 → 有效性校验 → 预览编辑 → 导出/加入题库
```

**流程详解**：

#### 步骤 1：上传原题截图
- **入口位置**：对话界面增加"📷 上传原题生成相似题"按钮
- **支持格式**：JPG、PNG、HEIC（手机拍照）
- **图片要求**：清晰、完整包含题目文字

#### 步骤 2：OCR 识别题目
- **技术选型**：**阿里云教育场景 OCR**
  - **成本**：¥0.001/次（约 $0.00014）
  - **准确率**：98%+（专门针对试题优化）
  - **速度**：1-2 秒
  - **支持内容**：印刷体文字 + 数学公式（LaTeX 格式）+ 表格识别
  - **备份方案**：**PaddleOCR**（开源免费，API 调用失败时本地兜底）

- **识别内容**：
  - **题目文字内容**（必须）
  - 数学公式（LaTeX 格式输出）
  - 选项（如有，大部分原题截图仅包含题目）
  - 答案（如有，通常不包含）
  - 解析（如有，通常不包含）

- **识别输出**：结构化 JSON 格式
  ```json
  {
    "question": "题目内容",
    "formulas": ["x^2 + y^2 = r^2"],  // LaTeX 格式公式
    "options": [],  // 通常为空，原题一般只有题目文字
    "answer": null,  // 通常为空
    "analysis": null,  // 通常为空
    "ocr_confidence": 0.98  // OCR 识别置信度
  }
  ```

**OCR 方案选择逻辑**：
1. 优先使用阿里云教育 OCR（准确率高、成本极低、专门针对教育场景）
2. 若阿里云 API 调用失败，降级到 PaddleOCR（本地识别兜底）
3. 识别成功后，**直接调用 Skill 生成相似题**（Skill 内部自动推断参数）

#### 步骤 3：生成相似题
- **调用 Skill**：`similar-question-generation`
- **输入内容**：OCR 识别结果（题目文字 + 公式）+ 生成数量（默认3道）
- **生成数量**：
  - **默认**：3 道（适合大部分场景）
  - **可动态调整**：1-10 道（根据用户需求）
  - **参数名**：`question_count`
- **Skill 内部处理**：
  - 自动推断学科、年级、知识点、题型、难度
  - 基于推断结果生成相似题
  - **单次 LLM 调用生成多道题**（JSON 数组格式）
- **生成规则**：
  - **核心层（0% 变化）**：知识点、数学模型、解题逻辑、能力考查层级 **完全锁定**
  - **结构层（≤10% 变化）**：至少改变 1 处（设问角度/求解对象/干扰项类型）
  - **表达层（30-50% 变化）**：场景完全陌生化（如环形跑道→公交环线→工程建设）
  - **数值层（80-100% 变化）**：具体数字、数量级大幅改变
- **输出格式**：每道题包含题目、选项（如有）、答案、解析
- **生成时长**：
  - 1道：约 20 秒
  - 3道：约 50 秒
  - 5道：约 80 秒
  - 10道：约 150 秒
  - *（耗时主要受 DeepSeek v4-flash 生成速度限制：46 tokens/秒）*

#### 步骤 4：有效性校验
- **校验目标**：确保生成的题目可解且答案唯一
- **校验内容**：
  - **条件充分性**：题目提供的信息足以求解
  - **逻辑一致性**：题目不包含矛盾条件
  - **答案唯一性**：解题路径明确，答案唯一确定
  - **知识点匹配**：题目确实考查目标知识点
- **校验方式**：LLM 推理验证（DeepSeek-V4）
- **失败处理**：
  - 单题校验失败：自动重新生成该题（最多重试 2 次）
  - 多次失败：提示用户"该题目不适合生成相似题"
- **校验时长**：约 5 秒（3 道题并行校验）

#### 步骤 5：预览编辑界面
- **布局方式**：
  ```
  ┌─────────────────────────────────────────────┐
  │  原题（左侧 30%）        │  相似题列表（右侧 70%）     │
  │  ┌──────────────┐      │  ┌──────────────────┐  │
  │  │ [原题截图]     │      │  │ 相似题 1          │  │
  │  │ 题目内容显示   │      │  │ [题目内容]        │  │
  │  │              │      │  │ [选项 A/B/C/D]    │  │
  │  └──────────────┘      │  │ [答案] [解析]     │  │
  │                        │  │ [✏️编辑] [🔄重新生成] │
  │                        │  └──────────────────┘  │
  │                        │  ┌──────────────────┐  │
  │                        │  │ 相似题 2          │  │
  │                        │  └──────────────────┘  │
  │                        │  ┌──────────────────┐  │
  │                        │  │ 相似题 3          │  │
  │                        │  └──────────────────┘  │
  └─────────────────────────────────────────────┘
  ```
- **操作功能**：
  - 单题编辑（修改文字、选项、答案）
  - 单题重新生成
  - 单题删除
  - 全部重新生成

#### 步骤 6：导出/加入试卷
- **操作选项**：
  - **加入当前试卷**：将 3 道相似题追加到当前正在编辑的试卷
  - **导出为新试卷**：单独导出为 Word 文档
  - **保存到题库**：将 3 道题保存到个人题库（后续功能）

### 2.4 数据模型

#### SimilarQuestionRequest（请求）

```typescript
interface SimilarQuestionRequest {
  original_question: {
    question: string;        // 题目内容
    options?: string[];      // 选项（可选）
    answer?: string;         // 答案（可选）
    analysis?: string;       // 解析（可选）
    image_url?: string;      // 原题截图 URL
  };
  params: {
    subject: string;         // 学科
    grade: string;           // 年级
    knowledge_point: string; // 知识点
    question_type: string;   // 题型
    difficulty: string;      // 难度
    similarity_level: 0.7;   // 相似度（固定 70%）
    count: 3;                // 生成数量（固定 3）
    change_dimensions: {     // 变化维度
      scene: boolean;        // 改变场景
      value: boolean;        // 改变数值
      expression: boolean;   // 改变表述
      question_angle: boolean; // 改变设问角度
    };
  };
}
```

```typescript
interface SimilarQuestionResponse {

  };

interface Question {

  explanation: string;
  metadata: {
    subject: string;
```

## 3. 交互流程详解

### 3.1 主流程

```
1. 用户点击"📷 上传原题生成相似题"

   ↓
4. 跳转到相似题预览界面
```

### 3.2 边界处理

#### 场景 1：OCR 识别失败
**现象**：图片模糊、手写字难识别、复杂公式识别错误  
**处理**：
1. 提示用户"识别失败，请重新上传清晰图片"
2. 支持手动输入题目内容
3. 提供"识别结果确认"步骤，用户可编辑识别出的文字

#### 场景 2：参数推断不准确
**现象**：学科、年级、知识点推断错误  
**处理**：
1. 生成的相似题会偏离预期（如题型不对、难度不匹配）
2. 用户可通过"全部重新生成"重新尝试
3. 后续版本可增加参数微调功能

#### 场景 3：生成的相似题质量不佳
**现象**：与原题过于相似、场景未陌生化、干扰项不合理  
**处理**：
1. 单题重新生成（不影响其他 2 道）
2. 全部重新生成
3. 手动编辑调整

#### 场景 5：有效性校验失败
**现象**：生成的题目条件不足、逻辑矛盾、无解或多解  
**处理**：
1. 自动重新生成该题（最多重试 2 次）
2. 多次失败后提示"该题目不适合生成相似题"
3. 建议用户尝试其他原题

#### 场景 4：原题本身不适合生成相似题
**现象**：纯记忆题（如"π 的值是多少"）、开放题（如作文）  
**处理**：
1. OCR 识别后自动判断题型
2. 不适合的题型弹窗提示"该题型不适合生成相似题"
3. 给出替代建议（如"可以尝试'智能出题'功能"）

## 5. 技术要点

### 5.1 技术选型

**OCR 识别**：阿里云教育场景 OCR
- **成本**：¥0.001/次（约 $0.00014）
- **准确率**：98%+（专门针对教育场景优化）
- **速度**：1-2 秒
- **优势**：
  - 专门针对试题优化（识别准确率高）
  - 支持数学公式识别（LaTeX 格式输出）
  - 成本极低（比 LLM Vision API 低 95%+）
  - 稳定性高（阿里云基础设施）
- **备份方案**：PaddleOCR（开源免费，API 调用失败时本地兜底）

**相似题生成**：复用蜜蜂家校 `similar-question-generation` Skill
- **输入**：OCR 识别结果（题目文字 + 公式）
- **Skill 内部处理**：
  - 自动推断学科、年级、知识点、题型、难度
  - 基于推断结果生成相似题
- **相似度**：70%（固定）
- **生成数量**：3 道（固定）
- **规则**：四层控制模型（核心层 0%、结构层 ≤10%、表达层 30-50%、数值层 80-100%）
- **LLM 调用**：DeepSeek-V4

## 7. 风险与挑战

### 7.1 技术风险

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| **OCR 识别准确率低** | 生成的相似题基于错误原题 | 1. 使用阿里云教育 OCR（准确率 98%+，专门针对试题优化） <br> 2. 识别结果展示给用户确认 <br> 3. 支持手动编辑识别结果 <br> 4. API 调用失败时降级到 PaddleOCR 本地识别 |
| **相似题质量不稳定** | 用户不满意，频繁重新生成 | 1. 严格遵守 Skill 规则（四层模型） <br> 2. 增加质量检查节点（逻辑性、可作答性） <br> 3. 支持单题重新生成 |
| **生成耗时过长** | 用户等待焦虑 | 1. 使用流式输出（每生成 1 道实时显示） <br> 2. 显示进度条和预计剩余时间 <br> 3. 支持后台生成（生成完成后通知） |
| **手写题识别困难** | 无法识别手写试卷 | 1. V1.0 仅支持印刷体 <br> 2. V1.1 优化手写识别（微调 OCR 模型） |

### 7.2 产品风险

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| **用户上传非题目图片** | 识别失败、浪费 API 成本 | 1. 图片上传前校验（检测是否包含文字） <br> 2. 识别失败后提示"请上传题目图片" |
| **相似题过于相似** | 学生"套格式"秒选答案 | 1. 严格执行场景陌生化（70% 相似度） <br> 2. 结构层至少改变 1 处 |
| **相似题偏离原题** | 失去"相似题"意义 | 1. 核心层锁定（知识点、模型、逻辑） <br> 2. 增加相似度检查节点 |
| **功能入口不明显** | 用户不知道有这个功能 | 1. 对话界面突出"📷 上传原题"按钮 <br> 2. 新用户引导动画 <br> 3. 首次使用弹窗说明 |

### 7.3 合规风险

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| **版权问题** | 用户上传教辅书籍原题 | 1. 仅生成相似题，不存储原题 <br> 2. 用户协议明确"仅供个人教学使用" |
| **题目质量问题** | AI 生成的题目有知识性错误 | 1. 增加答案验证节点（逻辑性检查） <br> 2. 提示用户"AI 生成，仅供参考，请人工审核" |
