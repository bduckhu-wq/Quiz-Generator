## 🌳 决策树A：课后作业场景（homework）

```
scene = homework
  ↓
检查内容入口是否缺失
  ├─ [章节 OR 知识点已提供]
  │   ├─ 用户提供章节（chapter）
  │   │   ↓
  │   │   使用路径A（教材章节）
  │   │   ↓
  │   │   检查教材版本（textbook）
  │   │     ├─ 已提供 → 内容入口完整 ✅
  │   │     └─ 未提供 → 追问："使用的教材版本？[人教版][北师大版]..."
  │   │
  │   └─ 用户提供知识点（knowledgePoints）
  │       ↓
  │       使用路径B（知识点）
  │       ↓
  │       内容入口完整 ✅
  │
  └─ [章节 AND 知识点都缺失]
      ↓
      追问："作业内容？
            📕 按教材章节（如：第16章）
            💡 按知识点（如：二次根式化简）"
      ↓
      用户选择 → 回到"检查内容入口是否缺失"
  ↓
内容入口确定 ✅
  ↓
应用场景策略：
  - 难度分布 = {简单:50%, 中等:40%, 困难:10%}
  - 题型分布 = {选择:30%, 填空:30%, 解答:30%, 其他:10%}
  - 题量 = 10 题（默认）
  ↓
生成参数确认面板
```

**关键特点**：
- ✅ 灵活双入口，不限制用户选择
- ✅ 无强制规则，以用户输入为准

## 🌳 决策树B：单元测验场景（unit_test）

```
scene = unit_test
  ↓
检查内容入口是否缺失
  ├─ [章节已提供]
  │   ↓
  │   使用路径A（教材章节）
  │   ↓
  │   检查教材版本（textbook）
  │     ├─ 已提供 → 内容入口完整 ✅
  │     └─ 未提供 → 追问："使用的教材版本？"
  │
  ├─ [知识点已提供，但章节缺失]
  │   ↓
  │   尝试反向映射：知识点 → 章节
  │   ↓
  │   查询 knowledge_to_chapter 表
  │     ├─ 映射成功（找到对应章节）
  │     │   ↓
  │     │   AI："已匹配到以下章节，请确认教材版本：
  │     │         📕 人教版 - 第16章（二次根式）
  │     │         📘 北师大版 - 第6章（二次根式）
  │     │         ✏️ 仍按知识点出题（不指定章节）"
  │     │   ↓
  │     │   用户选择：
  │     │     ├─ 选择教材版本 → 使用路径A（转为章节）✅
  │     │     └─ 选择"按知识点" → 使用路径B ✅
  │     │
  │     └─ 映射失败（未找到对应章节）
  │         ↓
  │         AI："未找到与'XXX'对应的教材章节，
  │               是否按知识点出题？
  │               [是] [否，我来指定章节]"
  │
  └─ [章节 AND 知识点都缺失]
      ↓
      追问："测验范围？
            📕 教材章节（如：第16章）← 推荐
            💡 知识点（如：二次根式）"
      ↓
      用户选择 → 回到"检查内容入口是否缺失"
  ↓
内容入口确定 ✅
  ↓
应用场景策略：
  - 难度分布 = {简单:30%, 中等:50%, 困难:20%}
  - 题型分布 = {选择:30%, 填空:20%, 解答:40%, 其他:10%}
  - 题量 = 20 题（默认）
  ↓
生成参数确认面板
```

**关键特点**：
- ✅ 推荐路径A（教材章节）
- ✅ 支持知识点→章节的反向映射
- ✅ 不强制，允许纯知识点路径

## 🌳 决策树C：期中期末考试场景（exam）★强制路径A

```
scene = exam
  ↓
识别考试类型（examType）
  ├─ 关键词匹配："期中" → examType = "期中"
  ├─ 关键词匹配："期末" → examType = "期末"
  └─ 未识别 → 追问："是期中考试还是期末考试？"
  ↓
examType 确定 ✅
  ↓
检查内容入口
  ├─ [章节已提供]
  │   ↓
  │   使用路径A（教材章节）
  │   ↓
  │   检查教材版本（textbook）
  │     ├─ 已提供 → 跳到"检查学期"
  │     └─ 未提供 → 追问："使用的教材版本？"
  │
  ├─ [知识点已提供，但章节缺失]★冲突处理
  │   ↓
  │   尝试反向映射：知识点 → 章节
  │   ↓
  │   查询 knowledge_to_chapter 表
  │     ├─ 映射成功（找到对应章节）
  │     │   ↓
  │     │   AI："检测到您想考查'二次函数'和'三角形'，
  │     │         已为您匹配到以下章节：
  │     │           ✅ 第18章（三角形）
  │     │           ✅ 第19章（二次函数）
  │     │         
  │     │         期末考试建议覆盖更广的范围，是否需要补充？
  │     │           [仅考这两章]
  │     │           [补充为下学期全部（第16-20章）]← 推荐"
  │     │   ↓
  │     │   用户选择 → 确定章节范围 ✅
  │     │
  │     └─ 映射失败
  │         ↓
  │         AI："期中期末考试需要指定教材章节范围，
  │               请选择：
  │               📕 指定教材版本和学期（我来自动匹配）← 推荐
  │               ✏️ 手动输入章节范围"
  │
  └─ [章节 AND 知识点都缺失]
      ↓
      追问（不提供知识点选项）：
        "好的！还需要确认：
         1. 使用的教材版本？
            [人教版] [北师大版] [苏教版] [其他]
         2. 哪个学期的{examType}？
            🌸 上学期{examType}
            🌻 下学期{examType}"
      ↓
      用户选择：教材版本 + 学期
      ↓
      自动查询 textbook_semester_mapping 表
        查询条件：
          textbook = 用户选择的教材
          subject = 已知学科
          grade = 已知年级
          semester = 用户选择的学期
          examType = "期中" / "期末"
        ↓
        查询结果：chapters = ["第16章", "第17章", ...]
        ↓
        内容入口 = {
          content_type: "textbook_chapters",
          textbook: 用户选择的教材,
          semester: 用户选择的学期,
          examType: examType,
          chapters: 查询结果
        } ✅
  ↓
检查学期（semester）
  ├─ 已提供/已查询 → 跳过
  ├─ 可从年级推断（如"初二"默认推断当前学期）→ 自动填充
  └─ 无法推断 → 追问："哪个学期？[上学期][下学期]"
  ↓
内容入口确定（必须是路径A）✅
  ↓
应用场景策略：
  - 难度分布 = {简单:20%, 中等:50%, 困难:30%}
  - 题型分布 = {选择:25%, 填空:15%, 解答:50%, 其他:10%}
  - 题量 = 30 题（默认）
  ↓
生成参数确认面板（展示自动匹配的章节列表）
```

**关键特点**：
- 🔥 **强制使用路径A**（教材章节）
- 🔥 不允许纯知识点路径（必须反向映射到章节）
- 🔥 自动查询教材映射表，填充章节范围
- 🔥 追问时不提供"按知识点出题"选项

## 🌳 决策树D：考前复习场景（review）★推荐路径B

```
scene = review
  ↓
识别复习目标（可选）
  ├─ 关键词匹配："期中" → reviewTarget = "期中"
  ├─ 关键词匹配："期末" → reviewTarget = "期末"
  └─ 未识别 → reviewTarget = null
  ↓
检查内容入口
  ├─ [知识点已提供]
  │   ↓
  │   使用路径B（知识点）✅
  │   ↓
  │   跳到"应用场景策略"
  │
  ├─ [章节已提供]
  │   ↓
  │   使用路径A（教材章节）✅
  │   ↓
  │   检查教材版本（textbook）
  │     ├─ 已提供 → 内容入口完整 ✅
  │     └─ 未提供 → 追问："使用的教材版本？"
  │   ↓
  │   跳到"应用场景策略"
  │
  └─ [章节 AND 知识点都缺失]
      ↓
      提供智能推荐选项：
        "请选择复习策略：
         
         🔥 **高频考点**（推荐！）← 路径B
            根据历年{reviewTarget}考试统计：
            ☑️ 二次函数综合题（85%考试出现）
            ☑️ 三角形全等证明（78%考试出现）
            ☐ 一元二次方程应用（65%考试出现）
         
         ⚠️ **易错知识点**← 路径B
            学生普遍薄弱：
            ☑️ 二次根式化简（错误率42%）
            ☑️ 平行四边形判定（错误率38%）
            ☐ 反比例函数图像（错误率35%）
         
         🎯 **压轴综合题**← 路径B
            历年压轴题型：
            ☐ 动点问题（难度⭐⭐⭐⭐⭐）
            ☐ 函数与几何综合（难度⭐⭐⭐⭐⭐）
         
         ✏️ **自定义知识点**← 路径B
            直接输入想要复习的知识点
         
         📕 **按教材章节复习**← 路径A
            指定教材版本和章节范围
         
         请选择 1-5，或告诉我具体内容~"
      ↓
      用户选择：
        ├─ 1（高频考点）
        │   ↓
        │   查询 review_recommendation 表
        │     查询条件：
        │       reviewTarget = "期末"
        │       subject = 已知学科
        │       grade = 已知年级
        │       strategy = "高频考点"
        │     ↓
        │     查询结果：推荐知识点列表
        │   ↓
        │   AI 展示可多选列表（默认全选推荐项）
        │   ↓
        │   用户确认选择
        │   ↓
        │   内容入口 = {
        │     content_type: "knowledge_points",
        │     strategy: "高频考点",
        │     knowledge_points: 用户选择的知识点
        │   } ✅
        │
        ├─ 2（易错知识点）→ 同上逻辑，strategy = "易错知识点"
        ├─ 3（压轴综合题）→ 同上逻辑，strategy = "压轴综合题"
        ├─ 4（自定义知识点）
        │   ↓
        │   AI："请输入想要复习的知识点（用逗号分隔）："
        │   ↓
        │   用户输入知识点
        │   ↓
        │   内容入口 = {
        │     content_type: "knowledge_points",
        │     strategy: "自定义",
        │     knowledge_points: 用户输入的知识点
        │   } ✅
        │
        └─ 5（按教材章节复习）
            ↓
            追问："请指定：
                  1. 教材版本？[人教版][北师大版]...
                  2. 复习章节范围？（如：第16-18章）"
            ↓
            用户输入
            ↓
            内容入口 = {
              content_type: "textbook_chapters",
              textbook: 用户指定教材,
              chapters: 用户指定章节
            } ✅
  ↓
内容入口确定 ✅
  ↓
应用场景策略：
  - 难度分布 = {简单:10%, 中等:40%, 困难:50%}（聚焦重难点）
  - 题型分布 = {选择:20%, 填空:10%, 解答:60%, 其他:10%}（大题为主）
  - 题量 = 15 题（默认）
  - 特殊参数：is_review_mode = True（启用变式题推荐）
  ↓
生成参数确认面板
```

**关键特点**：
- 🔥 **推荐路径B**（知识点），提供三种智能推荐策略
- ✅ 允许路径A（按章节复习），不强制
- ✅ 复习策略带有数据支撑（如"85%考试出现"）

## 📊 场景与内容入口的映射矩阵

| 场景 | 路径A（教材章节） | 路径B（知识点） | 强制规则 | 反向映射 |
|-----|-----------------|---------------|---------|---------|
| **课后作业** | ✅ 支持 | ✅ 支持 | ❌ 灵活 | ❌ 不需要 |
| **单元测验** | ✅ 推荐 | ✅ 支持 | ❌ 不强制 | ✅ 支持（知识点→章节） |
| **期中期末** | ✅ **强制** | ⚠️ 仅作为补充（需反向映射） | ✅ **必须有章节** | ✅ 必须（知识点→章节） |
| **考前复习** | ✅ 允许 | ✅ **推荐** | ❌ 不强制 | ❌ 不需要 |

**关键规则**：
- ✅ 绿色 = 完全支持
- ⚠️ 黄色 = 有条件支持
- ❌ 红色 = 不支持/不需要

## 🔧 后端工具函数调用时机

| 工具函数 | 调用时机 | 输入参数 | 返回结果 |
|---------|---------|---------|---------|
| **query_chapter_range()** | 期中期末场景，用户选择教材版本+学期后 | textbook, subject, grade, semester, examType | chapters: list[str] |
| **map_knowledge_to_chapter()** | 单元测验/期中期末场景，用户提供知识点时 | knowledgePoint, subject, grade | {textbook: str, chapter: str}[] |
| **get_review_recommendation()** | 考前复习场景，用户选择智能推荐策略时 | strategy, subject, grade, reviewTarget | recommendations: list[dict] |

## 🎯 冲突处理速查表

| 冲突类型 | 示例 | 处理方式 |
|---------|------|---------|
| 期末考试+知识点 | "期末考试，考二次函数和三角形" | 反向映射到章节 → 提示补充范围 → 推荐全学期章节 |
| 课后作业+期末关键词 | "期末复习作业，第16章" | 场景=复习（不是exam），内容=第16章 |
| 复习+章节 | "复习第16-18章" | 允许，使用路径A |
| 单元测验+宽泛知识点 | "单元测验，考三角形" | 反向映射 → 匹配多个章节 → 让用户选择 |
