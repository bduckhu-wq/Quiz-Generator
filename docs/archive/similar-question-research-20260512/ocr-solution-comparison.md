## DeepSeek-V4 详细分析

### 优势

**1. 成本极低**
- **$0.001/图**，是 GPT-4V 的 **1/7.5**，是 Claude 的 **1/15**
- 1000 次/月仅需 **$1**（vs GPT-4V $7.5、Claude $15）

**2. 中文识别优秀**
- 国产模型，中文训练数据充足
- 对中文数学题的语义理解更准确
- 支持中文特有的表述方式（如"小明"、"一元一次方程"等）

**3. 速度快**
- 2-3 秒完成识别 + 参数提取
- 国内服务器延迟低（<100ms）

**4. 数学公式识别良好**
- 支持 LaTeX 格式输出
- 识别准确率 90%+（复杂公式可能需要人工校正）

**5. 与现有技术栈兼容**
- 已在项目中使用 DeepSeek 作为主力模型
- API 调用方式一致，无需额外集成

### 劣势

**1. 复杂公式识别略逊于 GPT-4V/Claude**
- 多行公式、矩阵、积分符号等复杂场景准确率 85-90%
- GPT-4V/Claude 可达 95%+

**2. 文档较少**
- 官方文档不如 OpenAI/Anthropic 详细
- 社区讨论较少，遇到问题排查困难

**3. 稳定性风险**
- 国产模型，可能面临限流、服务波动
- 需要备用方案（如通义千问-VL）

## 实施建议

### 1. 初期（V1.0 MVP）

**使用 DeepSeek-V4 单一方案**：
```python
# backend/services/ocr_service.py

async def recognize_question_image(image_url: str) -> dict:
    """
    OCR 识别题目图片
    
    使用 DeepSeek-V4 进行识别
    """
    client = DeepSeekClient()
    
    prompt = """
    识别图片中的题目内容，提取以下信息：
    1. 题目文字（包括选项，保留原格式）
    2. 答案（如有）
    3. 解析（如有）
    4. 推断：学科、年级、知识点、题型、难度
    
    输出 JSON 格式：
    {
      "question": "题目内容",
      "options": ["A. xxx", "B. xxx", "C. xxx", "D. xxx"],
      "answer": "B",
      "analysis": "解析内容",
      "subject": "数学",
      "grade": "初二",
      "knowledge_point": "一元一次方程",
      "question_type": "选择题",
      "difficulty": "中等",
      "confidence": 0.95
    }
    """
    
    response = await client.vision_call(
        image_url=image_url,
        prompt=prompt,
        model="deepseek-chat"
    )
    
    return response
```

### 2. 后续优化（V1.1+）

**可选增强**：
- 置信度判断：`confidence < 0.85` 时提示用户仔细检查
- 复杂公式检测：识别出积分、矩阵等符号时切换到 GPT-4V
- 用户反馈：记录识别错误案例，持续优化 prompt
