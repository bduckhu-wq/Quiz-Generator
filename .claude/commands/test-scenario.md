# 场景测试命令

用于快速验证核心用户流程是否符合 PRD 定义。

## 测试场景列表

### 场景 1：核心参数缺失（追问补全）
**PRD 参考**：3.2 节"场景一"

**测试步骤**：
1. 用户输入："帮我出一份数学试卷"
2. 预期 AI 行为：
   - ✅ 提取参数：学科=数学
   - ❌ 年级=缺失、内容入口=缺失
   - 🤖 批量追问："几年级的？出题内容？"

**验证点**：
- [ ] AI 是否一次性追问多个缺失参数（而非逐个追问）
- [ ] 追问文案是否友好（如"可以一起告诉我~"）

---

### 场景 2：最低门槛满足（展示参数面板）
**PRD 参考**：3.2 节"场景一"

**测试步骤**：
1. 用户输入："初二的，人教版第十六章到第十八章"
2. 预期 AI 行为：
   - ✅ 学科=数学、年级=初二、教材=人教版、章节=第16-18章
   - 🤖 推断：学期=下学期、场景=期末考试
   - ✅ 最低门槛满足 → 展示参数确认面板

**验证点**：
- [ ] 参数面板是否从输入框容器内向上展开
- [ ] 是否包含滑块参数（难度/题量/时长）
- [ ] 是否显示推断的参数（学期、场景）

---

### 场景 3：知识点宽泛输入（细化追问）
**PRD 参考**：3.2 节"场景三"

**测试步骤**：
1. 用户输入："帮我出一份初三物理关于力的题目，10道选择题"
2. 预期 AI 行为：
   - ⚠️ 知识点"力"过于宽泛
   - 🤖 追问细化："力包含多个子知识点，请选择..."
   - 展示子知识点：力的概念、力的测量、弹力、重力、摩擦力...

**验证点**：
- [ ] AI 是否识别出宽泛知识点
- [ ] 是否提供子知识点选择（标签形式）
- [ ] 是否支持"全部涵盖"选项

---

### 场景 4：试卷生成与预览
**PRD 参考**：3.1 节"⑤ 生成试卷"

**测试步骤**：
1. 参数确认后点击"确认并生成"
2. 预期 Agent 行为（思考过程）：
   - ✅ 加载出题规则 Skill
   - ✅ 场景策略匹配 → 期末考试 2:5:3
   - ✅ 知识点覆盖分析（第16-18章）
   - 🔄 设计试卷蓝图
   - 🔄 调用 generate_exam 工具
   - ⏳ 质量校验
3. 生成完成后自动切换为左右分栏布局

**验证点**：
- [ ] Agent 思考过程是否实时展示（SSE 流式）
- [ ] 是否包含 6 个思考阶段
- [ ] 生成完成后是否自动切换为分栏视图

---

### 场景 5：题目改编
**PRD 参考**：3.1 节"⑦ 题目改编"

**测试步骤**：
1. 在试卷预览页 hover 某题 → 点击"✏️ 改编"
2. 弹出改编面板，选择"⬆️ 提高难度"
3. 点击"确认改编"
4. 预期 AdaptAgent 行为：
   - 分析原题难度
   - 调用 adapt_question 工具生成新题
   - 返回改编后的题目 JSON

**验证点**：
- [ ] 改编面板是否包含 6 种改编方向
- [ ] 改编后题目是否自动替换原题
- [ ] 题号、分值是否保持不变

---

## 执行方式

```bash
# 运行单个场景测试
/test-scenario 1

# 运行全部场景测试
/test-scenario all

# 输出测试报告（Markdown 格式）
/test-scenario report
```
