## 🎯 关键特性说明

### 1. 智能路由机制
```
传统关键词匹配 ❌              LLM语义理解 ✓
───────────────              ─────────────
"出题" → exam_skill           "帮我出份试卷" ✓
"出3道题" → ❌ 无匹配          "来份数学卷子" ✓
"随便搜搜" → ❌ 错误路由       "课后练习5题" ✓

优势:
• 语义理解，无需精确关键词
• 100%测试通过率（7/7）
• 支持各种口语化表达
```

### 2. 参数提取优化
```
┌─────────────────────────────────────────────────────┐
│ 输入: "三年级数学下学期第三章第二节课后练习5题"      │
│                                                       │
│ 提取结果:                                            │
│ ✓ subject: "数学"                                    │
│ ✓ grade: "三年级" （区分"初三"）                     │
│ ✓ knowledge_points: ["第三章第二节"] （完整章节）     │

### 3. 章节与知识点分离
```
数据库结构:

│   Question   │

       │
       ▼

├────────────────┤

```

### 4. 题目分配算法
```
用户指定5题 → 智能分配算法

┌─────────────────────────────────────┐
│ 场景默认配置:                        │
│  - 题型: 选30% 填40% 解30%           │
│  - 难度: 简50% 中40% 难10%           │
│  - 总量: 12题                        │
└─────────────────────────────────────┘
         │
         ▼ 用户覆盖: question_count = 5
┌─────────────────────────────────────┐
│ 分配计算:                            │
│  1. 按比例初分配                     │
│     选择: 5×30%≈2 填空: 5×40%≈2     │
│     解答: 5×30%≈2                    │
│                                      │
│  2. 难度细分                         │
│     选择: {简1, 中0, 难0}            │
│     填空: {简1, 中1, 难0}            │
│     解答: {简1, 中1, 难0}            │
│                                      │
│  3. 校验总数: 1+1+1+1+1 = 5 ✓        │
└─────────────────────────────────────┘

结果: 精准控制题量，无超量/不足问题
```

## 🔧 核心技术栈

```

├──────────────────────────────────────────────────┤
│  策略层    │ OpenClaw Skills System (Markdown)   │

│  ──────────┼────────────────────────────────────│
